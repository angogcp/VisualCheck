\## ① 最小構成（PoC用・低コスト）



\### 必須ハード（安く揃える）



\- Webカメラ（フルHDで十分）

&nbsp;   

\- 三脚 or 固定治具（ブレ防止）

&nbsp;   

\- LED照明（リング or バー型）

&nbsp;   

\- 普通のWindows PC / 小型PC

&nbsp;   



👉 \*\*ポイント\*\*



> カメラ性能より「固定」と「光」が9割



---



\## ② 撮影環境の作り方（ここが一番重要）



\### カメラ位置



\- 上から or 斜め45度

&nbsp;   

\- ケーブル先端が\*\*画面中央に来る\*\*

&nbsp;   



\### 背景



\- 完全に単色（黒 or 白）

&nbsp;   

\- 反射しない素材

&nbsp;   



\### 照明



\- 影が出ない

&nbsp;   

\- 直接照らさず「拡散」

&nbsp;   



📌 \*\*NG例\*\*



\- 蛍光灯だけ

&nbsp;   

\- 日光が入る

&nbsp;   

\- 作業者が動かすたびに位置ズレ

&nbsp;   



---



\## ③ 検査の考え方（AIに全部やらせない）



\### フェーズ分解



1\. \*\*形が決まっているか\*\*  

&nbsp;   　→ ルール（長さ・位置）

&nbsp;   

2\. \*\*見た目がおかしくないか\*\*  

&nbsp;   　→ AI（キズ・はみ出し）

&nbsp;   

3\. \*\*グレーゾーン\*\*  

&nbsp;   　→ 人

&nbsp;   



👉 これで精度もコストも下がる



---

```

Webカメラ
```

\## 🧰 1) そのまま現場PoC用：視覚検査パイプライン系



\### \*\*📍 visual\_inspection\*\*



\- Pythonベースのフレームワークで、\*\*Webカメラでも動く「自動目視検査」\*\*のサンプルあり

&nbsp;   

\- Windows / macOS / Linux に対応しているので現場PCで動かせる

&nbsp;   

\- カメラ取得 → 前処理 → 判定までの流れが詰まっている (\[GitHub](https://github.com/topics/visual-inspection?utm\_source=chatgpt.com "visual-inspection · GitHub Topics"))

&nbsp;   



👉 PoCとしてすぐ動かして、判定ロジックの検証に使える



---



\## 🧠 2) 自動検査＋エッジAI（組立て・量産向け）



\### \*\*📍 Defect Detection Application (DDA)\*\*



\- 工場向けの\*\*リアルタイム検査フロー\*\*として設計されたGitHubサンプル

&nbsp;   

\- Edge（現場端末）でディープラーニングモデルを動かせる構成

&nbsp;   

\- REST API などで他システムとの連携にも対応可能 (\[GitHub](https://github.com/awslabs/DefectDetectionApplication?utm\_source=chatgpt.com "Defect Detection Application - A Edge Based Industrial ..."))

&nbsp;   



👉 全体像（撮影 → 推論 → 結果配信）を学べるテンプレ



---



\## 🧠 3) データラベリング・学習補助



\### \*\*📍 CVAT（Computer Vision Annotation Tool）\*\*



\- ラベル付け用ブラウザUI付きツール

&nbsp;   

\- 品質検査用データのラベリングに最適

&nbsp;   

\- OK/NGラベル、欠陥箇所のBounding Box/Maskが付けられる (\[ウィキペディア](https://en.wikipedia.org/wiki/Computer\_Vision\_Annotation\_Tool?utm\_source=chatgpt.com "Computer Vision Annotation Tool"))

&nbsp;   



👉 教師データを整えるにはほぼ必須ツール



---



\## 🧠 4) モデル構築・強化学習



これらは\*\*品質検査モデル構築の基盤ライブラリ\*\*



\### \*\*📍 MediaPipe\*\*



\- Google製のリアルタイム画像処理＆推論フレームワーク

&nbsp;   

\- Webカメラ音声・画像解析としても動作できる (\[ウィキペディア](https://en.wikipedia.org/wiki/MediaPipe?utm\_source=chatgpt.com "MediaPipe"))

&nbsp;   



👉 軽量でリアルタイム処理向き（例：欠陥がある秒数だけトリガー出す）



\### \*\*📍 Albumentations\*\*



\- 拡張データ増強ライブラリ（学習データを増やすのに強い） (\[ウィキペディア](https://en.wikipedia.org/wiki/Albumentations?utm\_source=chatgpt.com "Albumentations"))

&nbsp;   



👉 画像の回転・明るさ変化などサンプル増やしで精度アップ



---



\## 🧠 5) 参考に使えるDeep Learning系実装
